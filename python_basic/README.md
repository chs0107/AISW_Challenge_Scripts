# 온라인 커뮤니티 신고·제재 시스템 시뮬레이터 (파이썬 기초 과목)

## 주제 선정 이유 (개연성)
우리 팀의 인공지능 과목 최종 서비스는 "커뮤니티 서버와 연결해 남녀차별
게시물을 감지하면 블라인드 처리하는 자동 필터링 봇"이다. 파이썬 기초
과목에서는 AI 모델(딥러닝 분류기)을 붙이기 전 단계로, **신고가 누적됐을
때 게시물과 사용자를 어떻게 제재할지에 대한 로직의 뼈대**를 객체지향적으로
먼저 설계하고 검증하는 프로토타입을 만들었다. 즉 이 프로젝트는 인공지능
과목 결과물의 "제재 처리 엔진" 부분을 파이썬 기초 단계에서 미리 구현해본
것으로, 팀 프로젝트 전체 스토리(남녀차별 문제 대응)와 자연스럽게 연결된다.

## 실행 방법
```
python3 moderation_sim.py
```

## 클래스 구조 및 OOP 개념 매핑

| 개념 | 코드 위치 | 설명 |
|---|---|---|
| 캡슐화 | `User._violation_count`, `_is_blocked` | 밑줄(_)로 직접 접근을 지양하고, `receive_violation()` 메서드를 통해서만 값을 안전하게 변경 |
| 상속 | `User` → `RegularUser`, `RepeatOffenderUser` / `ModerationBot` → `StrictBot`, `LenientBot` | 공통 로직은 부모 클래스에 한 번만 작성하고, `super().__init__()`으로 재사용 |
| 다형성 | `receive_violation()`, `review()` | 같은 이름의 메서드를 호출해도 사용자 유형/봇 종류에 따라 실제 동작(기준치)이 다르게 실행됨 |
| 클래스 간 관계 | `Post.author`(User 객체 포함), `CommunityServer`(Post·User·ModerationBot을 종합 관리) | 서로 다른 클래스의 객체가 속성으로 서로를 참조하며 협력 |

## 파일 구성
- `moderation_sim.py` : 전체 클래스 정의 + 시뮬레이션 실행 코드 (단일 파일, 초보자용 상세 주석 포함)

## 실행 시나리오
1. 일반 사용자(A)와 반복 위반자(B)가 각각 게시물을 작성
2. 엄격봇(신고 2건 기준)이 게시물을 검토하며 신고가 누적됨에 따라 블라인드/사용자 제재 처리
3. 동일한 신고 상황(2건)에 대해 엄격봇 vs 관대봇의 처리 결과를 비교하여 다형성 효과를 직접 확인
